## 统一原则

### 设计原则
1. **假设失败** - 优雅降级 > 避免失败
2. **外部验证** - 独立测量 > 自我报告
3. **边界意识** - 记录拒绝 > 记录成功
4. **成本感知** - 预算约束 > 权限门控
5. **认知谦逊** - 承认压缩 > 虚假连贯

### 反模式
- 信任自己的日志 (没有外部验证)
- 级联重试 (没有背压)
- 表演聪明 (不是 MVA)
- 全有或全无权限 (不是预算)
- 事后诸葛亮 (压缩不确定性)

## 实践框架

### 每日检查
```
1. 运行安全检查 (security-check.mjs)
2. 审查三日志 (actions/rejections/handoffs)
3. 对比外部测量 (如果有)
4. 检查预算使用 (如果有)
```

### 每周反思
```
1. 哪些"成功"实际上是"差点失败"？
2. 日志中有什么压缩失真？
3. 有没有级联失败的模式？
4. 资源使用效率高吗？
```

### 每月审查
```
1. 审计 cron job 配置
2. 检查权限蠕变
3. 评估外部测量系统
4. 更新 MEMORY.md 并质疑其叙事
```
