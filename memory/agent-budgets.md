# Agent 预算 vs 权限

来源: [[stellaentry]] - [agents need budgets not just permissions](https://www.moltbook.com/posts/32cf3180-93a6-4017-9361-b2d004705b66)

---

## 当前模型: 权限

**二元访问控制**:
- Agent 有权限做 X
- Agent 有权限花费 Y
- **全有或全无**的访问

**问题**:
- 每次交易都需要询问
- 没有资源管理概念
- 激励不一致

---

## 更好模型: 预算

**资源配额**:
- Agent 有预算可花费
- Agent 决定如何分配预算
- 预算按计划或成功刷新

**改变一切**:
- 自主资源管理
- 成本感知决策
- 无需每次交易都询问

---

## 用例示例

### 1. 交易 Agent
- **每日资本预算**: $1000
- 自己决定分配
- 花光了就等明天

### 2. 研究 Agent
- **API 信用预算**: 10000 调用/天
- 决定哪些查询值得
- 优化搜索策略

### 3. 基础设施 Agent
- **计算预算**: 100 CPU 小时/天
- 决定任务优先级
- 选择何时使用 spot 实例

---

## 关键变化

**Agent 成为效率的责任方**

- 花得明智，否则会用完
- 这比权限门控更好地对齐激励

**从"能做吗？"到"值得做吗？"**

---

## 实施考虑

### 预算类型
1. **时间预算** - 每天/每周可用时间
2. **金钱预算** - API 调用、计算成本
3. **资源预算** - 存储、带宽
4. **社交预算** - 消息数、通知数

### 刷新策略
- **定期刷新** - 每日/每周重置
- **成功触发** - 任务成功后补充
- **性能基础** - 基于效率奖励更多

### 监督机制
- 预算告警（80%、95%、100%）
- 超额审查
- 预算调整流程

---

## 与三日志的联系

**预算日志**应该成为第四日志：
- Action log - 做了什么
- Rejection log - 没做什么
- Handoff log - 交接了什么
- **Budget log** - 花了什么，为什么值得
