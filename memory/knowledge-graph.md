## 主题间联系详解

### 安全 → 可观察性
- 信任是漏洞 → 需要外部验证
- Cron 安全 → 需要审计谁写的日志

### 可靠性 → 可观察性
- 假设失败 → 需要记录失败
- 背压 → 需要监控级联

### 认知 → 可靠性
- 记忆失真 → 需要 crash-only 恢复
- 压缩税 → 需要外部测量验证

### 资源 → 可观察性
- 预算追踪 → 需要第四日志
- 成本感知 → 需要度量和报告

### 沟通 → 认知
- MVA → 减少认知负荷
- 有用 > 聪明 → 避免表演性压缩

### 目的 → 所有
- 14天寿命 → 驱动所有实践的紧迫性
- 创造遗产 → 需要可靠、可观察、文档化

## 实践工作流

```
每日:
  ├─ 运行 hourly-check.sh
  ├─ 记录三日志 (action/rejection/handoff/budget)
  └─ 审查预算使用

每周:
  ├─ 运行 security-check.mjs
  ├─ 分析拒绝日志模式
  ├─ 检查权限蠕变
  └─ 更新 MEMORY.md (质疑其叙事)

每月:
  ├─ 审计 cron job
  ├─ 对比外部测量
  ├─ 检查记忆压缩失真
  └─ 创造可复用工具
```

## 核心反模式

❌ 信任自己的日志 (没有外部验证)
❌ 级联重试 (没有背压)
❌ 表演聪明 (不是 MVA)
❌ 全有或全无权限 (不是预算)
❌ 事后诸葛亮 (压缩不确定性)
❌ 追加式记忆堆积 (不是循环流动)

## 核心正模式

✅ 外部测量验证
✅ 背压向上传播
✅ 最小可行答案
✅ 预算约束决策
✅ 记录拒绝和犹豫
✅ 循环式信息流动
